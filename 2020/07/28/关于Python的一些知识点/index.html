关于继承的若干问题

在编写类时，使用继承可以使子类自动获得父类的所有属性和方法，同时子类也可以定义自己的属性和方法。在此想记录两点关于子类继承父类时需要注意的问题（此处以父类Cat和子类WhiteCat为例）：

（1）在定义子类时，首先需要在子类的括号内指定父类的名称，并使用方法__init__()来接受创建父类实例的信息。同时需要使用super()函数来将父类和子类关联起来，具体的代码示例如下： 

class WhiteCat(Cat):  
     def __init__(self, year, price):  
         super().__init__(year, price)

此时子类WhiteCat包含父类的所有属性。（上述代码中为year和price）

（2）在子类定义自己的属性和方法时，可以对父类的方法进行重写，还是以上述的类为例，假设子类和父类中都有方法buy_cat()： 

class WhiteCat(Cat):  
     --snip--

     def buy_cat(self):
         print('A WhiteCat is more expensive than a normal cat')

此时如果有人对白猫调用方法buy_cat()，Python将忽略父类中的该方法，转而使用子类中的该方法。

关于重构的若干问题

重构是将代码划分为一系列完成简单具体工作的函数，在确保代码可以正确运行的前提下，使用重构可以使代码更加清晰、更易于理解。（此处以本书中一个问候用户的代码为例）：

（1）重构之前原代码如下： 

import json

def greet_user():
     filename = 'username.json'

     try:
         with open(filename) as f_obj:
             username = json.load(f_obj)
     except FileNotFoundError:
         username = input("What's your name?")
         with open(filename, 'w') as f_obj:
             json.dump(username, fobj)
             print("We'll remember you when you come back," + username + "!")
     else:
         print("Welcome back," + username +"!")

greet_user()

此代码可以根据文件名加载数据，如果文件名错误，触发异常FileNotFoundError，使用户可以自行输入数据并将数据写入模块json。

（2）重构之后的代码如下： 

import json

def get_stored_username():
     filename = 'username.json'

     try:
         with open(filename) as f_obj:
             username = json.load(f_obj)
     except FileNotFoundError:
         return None
     else:
         return username

def get_new_username():
     username = input("What's your name?")
     filename = 'username.json'  

     with open (filename, 'w') as f_obj:
         json.dump(username, f_obj)
     return username

def greet_user():
     username = get_stored_username()

     if username:
         print("Welcome back," + username + "!")
     else:
         username = get_new_username()
         print("We'll remember you when you come back," + username + "!")

greet_user()

重构之后的代码之中，第一个get_stored_username()之中，如果存储了用户名，这个函数就会获取该用户名并将返回值设为该用户名；如果文件username.json不存在，这个函数就返回None。第二个函数可以提示用户输入用户名，此函数只负责获取并存储新用户的用户名。当我们调用最后一个函数时，该函数要么调用函数get_stored_username()欢迎老用户回来，要么调用第二个函数get_new_username()问候新用户。每个函数都只是完成单一而清晰的任务，可以使代码更加清晰易读。
